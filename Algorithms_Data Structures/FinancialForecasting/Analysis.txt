Time Complexity:
Each recursive call decreases the number of years by 1.

T(n) = T(n−1) + O(1)

Therefore,
Time Complexity = O(n)
where n is the number of years.

Space Complexity:
Since each recursive call is stored in the call stack,
Space Complexity = O(n)


How to Optimize the Recursive Solution?

The recursive solution creates one function call for each year, increasing stack usage.

Optimization 1: Iterative Approach 
Replace recursion with a loop:
public static double futureValue(double currentValue, double growthRate, int years) {

    for (int i = 0; i < years; i++) {
        currentValue = currentValue * (1 + growthRate);
    }

    return currentValue;
}
Time Complexity: O(n)
Space Complexity: O(1)


Optimization 2: Mathematical Formula
Using the compound interest formula directly:
double futureValue = currentValue * Math.pow(1 + growthRate, years);
Time Complexity: O(1) (conceptually, for this assignment)
Space Complexity: O(1)

This is the fastest and most efficient method for calculating future value.

Comparison
Method	Time Complexity	Space Complexity
Recursive	O(n)	O(n)
Iterative	O(n)	O(1)
Mathematical Formula (Math.pow)	O(1) (conceptually)	O(1)
Conclusion

For a financial forecasting tool:
The recursive approach is useful for understanding recursion and demonstrates how a problem can be solved by repeatedly applying the same operation.
The iterative approach is more memory-efficient because it avoids recursive call stack overhead.
The mathematical formula (Math.pow) is the most suitable for production systems because it is concise, efficient, and avoids unnecessary computation.
