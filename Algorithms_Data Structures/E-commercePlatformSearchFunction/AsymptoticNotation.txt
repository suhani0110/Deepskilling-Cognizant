
Big O Notation -> Big O notation is a mathematical way to describe how the running time or memory usage of an algorithm grows as the input size increases.
It helps us:
        -> Compare different algorithms.
        -> Predict performance for large datasets.
        -> Choose the most efficient algorithm.

Search Operation Cases: 

1. Best Case

The element is found immediately.
Linear Search → First element → O(1)
Binary Search → Middle element → O(1)

2. Average Case

The element is somewhere in the middle.
Linear Search → O(n)
Binary Search → O(log n)

3.Worst Case

The element is at the last position or not present.
Linear Search → O(n)
Binary Search → O(log n)
